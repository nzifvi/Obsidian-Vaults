- Typecasting extends to objects.
	- By using this with subcasting, it allows for very useful outcomes.


```java
public class Parent{
	//Attributes
	//Methods
}

public class Child extends Parent{
	//Attributes
	//Methods
}


```

- Suppose you have a parent class and a parent's child class.
	- By using subtyping, you can assign a child object to an array of the parent class type.

```java
Parent[] array = new Parent[5];
Child c1 = new Child();
array[0] = c1;
```

- Despite being loaded into an array of parent classes, the child object has all of it's attributes and methods preserved. They can be later accessed using typecasting.


```java
c1Obj = (Child) array[0];
```

- Typecasting provides additional information to the compiler, telling the compiler what data type it is.
	- Provides access to methods that only exist in the child class.
	- Allows variables, that may have a value from when the object was assigned to the array, can be retrieved.
