[[3.1) Subtyping]]

---

### Liskov Substitution Principle (LSP):

If S is a subtype of T (child class of T) then objects of type T can be substituted (replaced) with objects of type S, without altering properties of the program.

### Consequence of LSP:

-  Subtype objects, of a supertype object, can be passed as parameters even if the specified parameter is the supertype:

```java

//Suppose A, B, C is a subtype of T:

public void print(T obj){
	System.out.println(obj.get());
}

public void print(T obj){
	T obj1 = new T();
	print(obj1);
	
	obj1 = new A();
	print(obj1);
	
	obj1 = new B();
	print(obj1);
	
	obj1 = new C();
	print(obj1);
}

```

- When assigning, or reassigning, a subtype to a supertype, the data type of the variable is not changed.
	- They are just STORED in a variable which has a datatype of the supertype.
