## What is Subtyping?

Subtyping means that, if an object of a super class is initialised, you can ASSIGN any subclass to that object.

```java

//suppose Vehicle is super class of Car

Vehicle v1 = new Car(); //Assigns car object, besides datatype being Vehicle.

Vehicle v1 = new Vehicle();
v1 = new Car(); //also works...



```

In Java, variables holding objects are Polymorphic.
- Can hold objects of more than one type.
- The declared type.
OR
- Subtypes of the declared type.
OR
- Reassigned to other subtypes. They must all extend from a common parent though.

NOTE THAT...
1) When subtyping, will only have access to methods parent has as well. You just can implement different behaviours of those methods.
	- Anything that is new in a subclass, that is not in the superclass, will not be available when subtyping.
	- To gain access to new methods or attributes, type casting is required.

2) A variable CANNOT be assigned sibling objects OR the parent object itself
```java
//suppose Van and Car are children of Vehicle

Car c1 = new Van(); //will error, due to sibling objects.

Car c2 = new Vehicle(); //will error, due to Van being parent of Car
```
