RULES FOR WHEN COMMENTING

1) Always attempt to explain yourself in code. Explain how it works.

2) Do not create redundant comments for something that is easy to understand.

3) Do not add comments for obvious things.

4) Do not comment out code, just remove it once it is no longer required.

5) Use as an explanation of intent.

6) Use to clarify code.

7) Use to warn of consequences if something happens.
