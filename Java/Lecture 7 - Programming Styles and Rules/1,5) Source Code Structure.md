RULES TO APPLY WHEN STRUCTURING SOURCE CODE

1) Separate concepts vertically.

2) Related code should appear "vertically dense" (close to each other.)

3) Declare variables close to where they are used, unless these are class attributes (declare class attributes at the top of the class).

4) Dependent functions should be close.

5) Similar functions should be close.

6) Place functions in the downward direction and, when adding new functions, keep going down.

7) Keep lines short.

8) Do not use horizontal alignment.

9) Use whitespace to associate related things and disassociate weakly related things.

10) Do not break indentation.
