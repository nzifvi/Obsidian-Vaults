It is possible for some try block to throw multiple types of Exceptions.
- You might want to do different things in a catch block based on what type of Exception was thrown.

instead of 1 catch block, which catches all Exceptions, also referred to as a polymorphic catch.
- Polymorphic catches means that all Exceptions are caught by it. Consequently, all Exceptions handled the same way.
- Not usually recommended.
```java
try{

}catch(Exception e){

}
```

You can have MULTIPLE catch blocks which each catch a specific Exception
- Each catch handles a new possible exception uniquely, allowing the program to handle the Exception gracefully.
- Program should be prevented from terminating if it is an Exception.
- This approach is preferred.

```java
try{
    //does something
}catch(FileNotFoundException e){
    //does something if this occurs.
}catch(SecurityException e){
    //does something if this occurs.
}
```
