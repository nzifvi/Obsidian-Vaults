Finally is an optional third block for the try-catch statement

```java
try{
    //code block tried
}catch(Exception e){
    //Exceptions caught
}finally{
    //Finally always executed, even if something was caught.
}
```

Even if a return statement is present within the try, or catch, parts of the statement, the finally is executed before the return occurs.

If a try completes, without any Exception being thrown, finally will be executed.
- Once finally is executed, if there is a return in try, the return is executed.

If a try throws an Exception, the try is stopped from executing and the catch executes. Once the catch executes, the finally will begin to execute.
- If the catch has a return, the finally will execute before the return is executed.


No matter what, the finally block is ALWAYS executed.


### When to use Finally?

Finally is often omitted but it is very valuable for creating clean programs where the try block might do something, external to the program, such as opening a file.

The finally part of a try-catch-finally statement can be used to clean up the actions of the try statement, ensuring the program state is always clean and nothing is left open or in memory.
