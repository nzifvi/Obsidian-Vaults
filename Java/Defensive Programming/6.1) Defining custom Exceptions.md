It is possible to define custom Exceptions that are to be thrown in the event that a program enters a state that would cause it to error.

- An Exception is mostly a class. It abstracts other things though.

- Since it is a class, you can create a child b y inheriting it.

```java
public class AuthenticationException extends Exception{
    private int reason;
    private static final int BAD_PW = 1;
    private static final int NO_USERNAME_FOUND = 2;
    public AuthenticationException(int reason){
	    this.reason = reason;
    }
}
```

- Might want to store variables which are reasons why this exception has been thrown.
	- Constructor for custom Exception takes in the actual reason it has been thrown this time

```java
    throw new AuthenticationException(1); 
    //throws AuthenticationException with reason as BAD_PW
```

- This allows for more context with error messages as there can be a method, called getReason(), which returns the reason.

- The catch block, responsible for handling this exception, can do different things based on this Exception.

```java
try{
    //suppose AuthenticationException thrown...
}catch(AuthenticationException e){
    if(e.getReason() == 1){
        //Can handle this reason specifically.
    }else if(e.getReason() == 2){
        //Can handle this other reason specifically.
    }
}
```
