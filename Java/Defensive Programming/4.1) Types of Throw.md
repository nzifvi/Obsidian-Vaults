## Error and Exception

Throwable, referred to as Throw, is a base class. It has 2 direct children:
1) Error.
2) Exception.

![[Pasted image 20250316181557.png]]
### What is an Error?

An error is known as a serious error. Most applications should not try to catch them.
- Errors are not checked.
- Most programs break during these situations.

Examples: VM error, failed JUnit assertion, incorrect syntax in source code.

### What are Runtime Exceptions?
RuntimeException, a child class of Exception, are unchecked.
- They are thrown during normal program operation (during runtime, after compilation).
- They are USUALLY caused by a program error.
- Can be avoidable through defensive program.
- Hard to recover from once they occur.

### What are Unchecked Exceptions?

Unchecked Exceptions are a child of  RunTimeException.

They are unchecked, meaning that the compiler does not force you to check for these with a try-catch.

If an Unchecked Exception occurs, they still have the be handled or passed up the call stack until they are caught with a try-catch OR the program exits due to them occurring.

UncheckedExceptions will still terminate the program if not handled.

Examples: ArrayOutOfBoundsException, NullPointerException.
