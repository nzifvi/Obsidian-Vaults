---
Defensive programming = coding practice that is used to anticipate, handle potential errors, and verify user input to prevent a program from terminating or performing unexpected, and potentially dangerous, program behaviour.

- Defensive programming makes a program more robust, reliable, and secure.

Defensive programming takes care of...
1) Passing null values
2) Constructor calls with bad parameters
3) Situations when a get method, from a collection (Queue, Dynamic Array,...).
4)  Situations where a user is trying to insert a duplicate record or a duplicate key.
5) Ensuring data is in the correct format.
	- Making sure a date is correct format.
	- Making sure a number is a correct/intended value.
	- In general, making sure some data meets some standard.

6) Check that an object state is consistent with an action requested (i.e. method called).
	- If a method, such as save(), requires something to be loaded, is it loaded? (usually check with booleans).
