When handling Exceptions, a try-catch is used:

```java
try{
    //try this block of code
}catch{
    //if an error occurs during the try block, handle here
}
```

In the catch block, the error can be handled
- Can set some variable to a value that prevents the error from happening.
- Can perform an initialisation if something failed to be read from a file, or another array/data structure.
- Etc etc.

You can also just give out an error message, plus a description, of what Exception was thrown.
- Describe what went wrong, where it went wrong, and any useful values.
- The catch block should NOT just report something has gone wrong. It should either return a value that signals an error has occurred OR it should fix the program state, by updating a variable or doing something, that allows the code after the try-catch to without an error occurring.

### PROCESS OF TRY-CATCH 

1) Execution moves into the try block statement.
   
2) If an exception is thrown, the execution IMMEDIATELY moves into the catch block: disregarding any following lines in the try block.
   If the error is handled, and catch does not have a return, then execution will continue after the try-catch statement.
   
3) If no exception is thrown during the try, catch is never executed.
   All lines during try will execute successfully.
