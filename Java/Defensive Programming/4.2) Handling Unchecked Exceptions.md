Two ways to handle...

1) Using try-catch

```java
try{
    //Try code that might throw unchecked exception
}catch(Exception e){
    //Handle
}
```

2) Test for something that will cause an unchecked exception in a method call that is about to happen

```java
if(i < 0 && i >= array.length){
	System.out.println("Please input a new integer");
}else{
    posFound = getItem(array, i);
}
```

Tests are often a better solution than a try-catch as you can still correctly send a correct value, or avoid whatever causes the unchecked exception, and get a correct output from the method.
