In the event that an exception is thrown, yet the program cannot do anything about it directly, an empty catch can be used.

```java
try{
    //something the program cannot fix occurs.
}catch(Exception e){
    //the program is prevented from terminating.
}
```

Catches are okay to be empty, as long as the following code, or the code which called the method that just threw an Exception, will not error as a result of the Exception being thrown.
