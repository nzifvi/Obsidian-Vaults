When programming, you should assume some things.

1) The other party (the party using the program) will not always know how to use a program, unless clearly told how to.
2) The other party (the party using the program) will not always do what they should do at all stages of a program.
3) If something can go wrong then it will go wrong.
	- If some block of code can go wrong then it should be made so that it handles what can go wrong gracefully there and then.
