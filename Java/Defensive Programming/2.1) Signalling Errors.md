[[3.1) Exceptions]] <- Next

Methods can return certain things to indicate that something has gone wrong, or some error has occurred during the method's operation.

When signalling an error, you can...
1) Return a value to indicate an error.
   Have to be confident it is not possible for a method to return a value that is considered an error when there is no error.
   
   For example, if an array stores a list of integers and you return an integer. You cannot use -1 as an error signal since there may be an element with a value of -1 in the list.
   
   If returning a primitive int, it is best to return the position, not the value, in the array.
   
   This means you can use an out of bounds value, such as -1, to signal an error as there can never be a -1 position in the array.
   
2) Return a boolean, instead of void, where true = some operation was successful and false = some operation was not successful.
   
3) Null, instead of an object, for failure.
   Use when an object is intended to be returned as you cannot return a primitive if so.
