### Error Message Audience

When an error occurs, some kind of message to inform those involved with the program.

- An error message should describe what has happened.
- Can provide additional info but be careful to which person this is presented to.

Parties that might want to see an error message from a program:
1) Author of code.
2) Another program, using your program, would potentially need a message of what went wrong.
3) The user, whose operation is affected.
	- Inform user of what caused error (bad input, invalid option, etc).


### Methods of notifying users

1) System.err.println(...)
	- A console message.
	- Console messages only available for people that look there AND have access to it.

2) Log file message
	- Saved to a folder which contains error log files.
	- People tend not to look there plus they may not even know it exists.

3) Dialogue box
	- A GUI which pops up when an error occurs.

When notifying a user of an error, give them an informative error message that tells them what went wrong.
- If a way to fix it exists, without having access to code, tell them (e.g. "input integer less than size of array).
