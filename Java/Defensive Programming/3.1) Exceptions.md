[[2.1) Signalling Errors]] <- Back

---

When encountering a value which is considered as a signal an error has occurred...
- Method returned null, or a value, which indicates something has gone wrong.
... Exceptions are used.

An Exception is thrown when an error is found.
- JVM throws exceptions itself, such as when you access an Out of Bounds Element in an Array.
- NullPointerExceptions.
- Many more.

A method exits with a throw, not the return value.

The code that called the method, which has thrown, has to handle the Exception.
- The responsible code can...
1) Catch and handle the exception there.
2) Pass the throw back to the code that called the code that called the method that threw.
- Continuing execution, disregarding the Exception, is not possible. The Exception MUST be handled some way or another.
