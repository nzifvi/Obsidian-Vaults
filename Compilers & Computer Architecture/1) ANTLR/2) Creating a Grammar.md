# What is a Grammar?
The grammar of a language specifies how the language works and what appears.

In ANTLR, the Grammar of a language is specified file with a .g4 file extension.
- ANTLR grammar file allows for a parser and lexer to be generated by specifying them.
- ANTLR grammar file is specified in the following order.

```ANTLR
// Parser Rules:

// Lexer Rules
```

# Parser Rules
The parser rules define the structure of the language
- Parser rules based on CFGs.

The first rule MUST be the start symbol.
```g4
Nonterminal : Production Rule;
```
- The terminals in the production rules are defined as token classes in the Lexer rules.

A nonterminal can have multiple production rules using the `|` symbol, which essentially means OR.
```g4
EXPR : NUM
     | EXPR + EXPR
     | EXPR - EXPR;
```
- Always finish with ;
# Lexer Rules
Lexer rules define the tokens and use RegEx to match strings, or part of a string, to certain token classes.

Each rule defines:
1) A token class.
2) A RegEx which denotes what strings can be part of the token class.

The token classes match to a terminal in the parser rules.

# Example
```g4
EXPR : EXPR '+' EXPR
     | EXPR '-' EXPR
     | EXPR '*' EXPR
     | EXPR '\' EXPR
     | NUM;
```
