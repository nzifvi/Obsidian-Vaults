## Signal/Flag Variables
Signal/Flag variables have one purpose. Their value is usually a boolean, or a binary value, and it used by some process to monitor if some kind of event has happened. It's value can be copied, or operated on, but it's primary purpose is to store some kind of indication that something has happened.

Just like any variable, the name of the signal/flag variable should explain what the signal/flag is used to indicate has occurred.
## Naming Convention for Signal/Flag Variables
Every character, in each word in a signal/flag variable, is capitalised and each word is separated with underscores.

In addition, affix the word SIGNAL to the front so it is immediately clear what it is. This is so that a long signal name does not hide the indication that it is a signal.

```c++
bool SIGNAL_QUEUE_HAS_ELEMENT;
bool SIGNAL_FILE_READY_TO_READ;
bool SIGNAL_FILE_CANNOT_BE_FOUND;
```

If the signal/flag is a pointer, append ptr to the front of the signal/flag variable name. Do not separate with an underscore.

```c++
bool ptrSIGNAL_PERFORM_WRITE;
```
