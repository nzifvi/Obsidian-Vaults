## What is Normalisation?
Normalisation is a process which transforms a flat-file database into a proper relational database by applying 3 transforms one by one.

Normalisation, if done correctly, ensures...
1) Each entity has the minimal number of attributes.
2) An entity contains ONLY attributes related to it.
3) Data redundancy is kept to the lowest it can possibly be.

Normalisation is referred to as a bottom-up approach.

Occurs over 3 transform stages, referred to as normal forms (NF).
1) 1NF
2) 2NF
3) 3NF

- Each stage requires a different process to be performed.
- Additional NFs do exist, however, it is unlikely they will be needed.
