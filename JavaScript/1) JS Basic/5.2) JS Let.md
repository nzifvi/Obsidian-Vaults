[[5.3) JS Const]]
- Variables declared with let have block scope.
- Variables declared with let MUST BE DECLARED before use.
- Variables declared with let CANNOT be redeclared in the same scope.

##  1) Block Scope:
- With the introduction of the keywords let and const, Block Scope was provided to JS.

- Variables declared inside of a { } block DO NOT EXIST, thus cannot be accessed, from outside the block.
```
{
	let x = 2;
}
// x does NOT exist outside the block it was declared in.
```

## 2) Global Scope:
- Variables declared with the var keyword ALWAYS have a Global Scope.
	- They CANNOT have a Block Scope.

```
{
	var x = 2;
}
//x DOES exist here, outside of the block it was declared and initialised in.
```

- Variables declared with var keyword, inside of a { } block, CAN BE ACCESSED from outside the block.

## 3) Redeclared / Not Redeclared:
- Variables declared with let CANNOT be declared. 
```
let x = "John Doe";
let x = 0; //ERROR WILL OCCUR DUE TO REDECLARATION OF A LET DECLARED VARIABLE.
```

- Variables declared and initialised with var CAN be redeclared.
```
var x = "John Doe";
var x = 0; //NO ERROR OCCURS.
```
