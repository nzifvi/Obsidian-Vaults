## Type Safety
EVERY object has a data type specified upon it being declared.

A program, or part of a program, is type-safe when objects are used ONLY by the rules of their datatype.
- However, not type-safe operations can be performed on objects.

An example of something not being type safe is using a variable before it has been initialised.
- NOTE: ALWAYS INITIALISE VARIABLES.

A program being completely type safe is the ideal state of a program.

NOTE: C++ compiler cannot guarantee complete type safety.

Type safety violations can be avoided by...
1) Good coding practice.
2) Run-time checks.
