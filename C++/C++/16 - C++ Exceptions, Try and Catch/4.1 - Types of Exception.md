## Types of Exception
There are 3 sets of exceptions...
1) Built-in Exceptions.
	1.1) Standard Exceptions.
2) User-defined Exceptions

## 1) Built-in Exceptions
C++, by default, provides a set of built-in exceptions. All built-in exceptions either directly or indirectly inherit from a base class: the exception class.
- exception class = root parent of all built-in exception types in C++

![[Pasted image 20250604132623.png]]

6 direct child classes of the exception class exist: 
- 2 are part of the standard exceptions set: run_time error and logic_error. They have their own child exception classes.
- There other 4 are part of the standard exception set BUT do not have any child classes: bad_alloc, bad_cast, bad_type_id, bad_exception

### Standard Exception Set
#### runtime_error
The run_time child class of exception has 2 child classes itself:
- overflow_error
	- Thrown when the result of an operation is larger than the largest number that can be stored within a datatype.
- underflow_error
	- Thrown when the result of an operation is smaller than the smallest number that can be stored within a dataype.

runtime_error child exception classes indicate that an error has occurred within a program's runtime

#### logic_error
The logic_error child class of exception has 3 child classes itself:
- invalid_argument
	- Thrown when an invalid argument was passed to a function.
- length_error
- out_of_range
	- Thrown when a value, used as an index, is larger than is not in the range of possible indices for an array data structure.

logic_error child exception classes indicate that an error has occurred within a program's logic.


### Other standard exceptions 

1) bad_alloc
	- bad_alloc is thrown if an attempt to allocate memory to the heap fails due to a lack of capacity.

2) bad_cast
	- bad_cast exception is thrown by dynamic_cast<>()

3) bad_type_id
	- bad_type_id exception is thrown by typeid

4) bad_exception

## 2) User-defined Exceptions
User-defined exceptions are exceptions made by a programmer.
- Made when the standard or non-standard exceptions do not accurately describe an error that has occurred.
